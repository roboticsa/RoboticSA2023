The race towards performance pushes robotic engineers to neglect software quality attributes of a robot controlsystem, such as maintainability, interoperability, scalability, and reusability. Software that is nicely encapsulated often simply runs slower than software that breaks encapsulation. Experience in software architecture design is about knowing when to break encapsulation to make the code run faster, and when not to do that.
